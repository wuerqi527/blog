Centos 服务器安装 Laravel Kafka

准备工作

安装软件

按照文档的顺序安装。

java
// yum 搜索 java
yum search java
yum install java-1.8.0-openjdk-devel.x86_64
zookeeper
// 下载
cd /usr/local/
wget https://mirrors.tuna.tsinghua.edu.cn/apache/zookeeper/zookeeper-3.4.14/zookeeper-3.4.14.tar.gz
// 得到 zookeeper-3.4.14
tar -zvxf zookeeper-3.4.14.tar.gz
// 将zookeeper后缀 移除
mv /usr/local/zookeeper-3.4.14 /usr/local/zookeeper
// 添加全局命令
vim /etc/profile
// profile文件尾部添加:
文件尾部添加：export ZOOKEEPER_HOME=/usr/local/zookeeper/
文件尾部添加：export PATH=$ZOOKEEPER_HOME/bin:$PATH
文件尾部添加：export PATH
// 发布刚刚修改的profile命令
source /etc/profile
// zookeeper 默认无配置文件，需要生成一个配置文件
cd /usr/local/zookeeper/conf
cp zoo_sample.cfg zoo.cfg
vim zoo.cfg
// 更改zoo.cfg配置文件中的dataDir (默认放至 /tmp/data) 以及dataLogDir (默认至 /tmp/data-log)
dataDir修改为： dataDir=/usr/local/zookeeper/data // 可先在目录下创建文件
dataLogDir修改为：dataLogDir=/usr/local/zookeeper/logs // 可先在目录下创建文件
// 运行命令
zkServer.sh start
// 查看zookeeper是否启动成功
// 若启动失败，修复文档：https://blog.csdn.net/cc1949/article/details/78196226
// 出错基本都是没有日志生成权限
zkServer.sh status

安装完成！

kafka
// 下载
cd /usr/local/
wget http://mirrors.shu.edu.cn/apache/kafka/2.1.1/kafka_2.11-2.1.1.tgz
tar -zvxf kafka_2.11-2.1.1.tgz
// 删除文件包
rm -rf kafka_2.11-2.1.1.tgz
mv /usr/local/kafka_2.11-2.1.1 /usr/local/kafka
cd kafka/config
// 更改kafka的服务器配置文件
log.dirs更改为：/usr/local/kafka/kafka-logs  // 放在kafka文件下
num.partitions更改为：4  // 这里随便给的

安装完成！
当前为1台broker的形式，多台需要简单配置一下！

librdkafka

librdkafka为kafka扩展

cd /usr/local/
mkdir librdkafka
git clone https://github.com/edenhill/librdkafka.git
./configure
make
make install
php-kafka
pecl install rdkafka

检测是否安装成功

php -i | grep rdkafka

安装文档地址：https://packagist.org/packages/rapide/laravel-queue-kafka

laravel 安装 kafka 扩展
composer require rapide/laravel-queue-kafka

配置使用地址：https://packagist.org/packages/rapide/laravel-queue-kafka

laravel-kafka 运行

开启kafka

cd /usr/local/kafka
bin/kafka-server-start.sh config/server.properties

kafka开启多个服务器（broker-0.properties为copy server.properties），修改broker id生成的

bin/kafka-server-start.sh config/broker0.properties
bin/kafka-server-start.sh config/broker1.properties
kafka配置本地broker注意事项
1.若添加多个broker需将/usr/local/celler/kafka/2.1.0/libexec/config/server.properties
2. 修改端口
3. 修改文件所在地址 /usr/local/var/lib/kafka-logs/meta.properties broker_id 设置成对应的id,然后启动服务
4. 复制 {KAFKA_PATH}/config/server.properties，需要模拟几台机器就复制几份
5. broker.id (文件开头) 必须唯一
6. listeners=PLAINTEXT://:9092 每一个文件的端口号必须唯一
7. log.dirs 文件可指定，指定路径为绝对路径

配置文档教程地址：https://blog.csdn.net/camel84/article/details/81879118

查看所有topic列表

bin/kafka-topics.sh –zookeeper localhost:2181 –list

查看指定topic明细

bin/kafka-topic.sh –zookeeper localhost:2181 –describe –topic {topic-name}

新建一个topic
bin/kafka-topics.sh --zookeeper localhost:2181 --create --topic {topic-name} --partitions {partition_name} --replication-factor {replication-factor}

1. {topic-name} topic名
2. {partition_name} 分区数
3. {replication-factor} 复制因子
本地模拟producer发送kafka消息
bin/kafka-console-producer.sh --broker-list localhost:9092 localhost:9093 localhost:9094 --topic {topic-name}
本地模拟consume接收kafka消息
bin/kafka-console-consumer.sh --bootstrap-server {server1} {server2} --topic {topic-name} --from-beginning
注意事项：
1.创建topic时，不可以使用 "." 或者 "_"，容易引发错误
2.kafka端口2181，broker端口为9092开头，不可混淆
3.获取一个文件内指定字符所在位置 grep localhost /etc/hosts
4.查找一个文件所在位置 find / -name 'hosts'

本地先使用测试数据实现kafka

首先定义一个 TestController.php，通过队列形式异步记录日志内容，代码如下：

<?php

namespace App\Http\Controllers\Api;

use App\Jobs\TestKafka;
use Illuminate\Http\Request;

class TestController extends AbstractController
{
    static $num = 0;

    public function index()
    {
        $string = self::$num + 1;

        $message = $string . ' :' . date('Y-m-d H:i:s', time()) . ' :' . rand(10000, 99999);

        TestKafka::dispatch($message);

        return ok(__FUNCTION__, compact('message'));
    }
}

我们创建一个TestKafka Job任务，往日志中写入内容，代码如下：

<?php

namespace App\Jobs;

use Illuminate\Bus\Queueable;
use Illuminate\Queue\SerializesModels;
use Illuminate\Queue\InteractsWithQueue;
use Illuminate\Contracts\Queue\ShouldQueue;
use Illuminate\Foundation\Bus\Dispatchable;

class TestKafka implements ShouldQueue
{
    use Dispatchable, InteractsWithQueue, Queueable, SerializesModels;

    protected $data;

    public function __construct(string $data)
    {
        $this->data = $data;
    }

    /**
     * Execute the job.
     *
     * @return void
     */
    public function handle()
    {
        \Log::channel('successLog')
            ->error('测试kafka', [
                'time' => date('Y-m-d H:i:s'),
                'date' => $this->data,
                '--------------------------------- data 分割 ---------------',
            ]);
    }
}

.env文件修改QUEUE_DRIVER

QUEUE_DRIVER=kafka

配置成功，然后我们需要开启队列服务，本地可使用命令

php artisan queue:work

但是报错，报错显示:

Configuration property offset.store.method is deprecated: Offset commit store method: 'file' - DEPRECATED: local file store (offset.store.path, et.al), 'broker' - broker commit store (requires Apache Kafka 0.8.2 or later on the broker).

这里报错是因为php-kafka支持的kafka版本为0.8，实际的kafka版本为1.1，需要修复一下，我们进入到项目目录
/vendor/rapide/laravel-queue-kafka/src/Queue/Connector/KafkaConnector.php，搜索‘offset.store.method’
将该行注销，再次运行将成功。
